Prototype change log

This summarises the main changes we’ve made in response to feedback.

Version 0 to 1

Version 0 was a baseline version based closely on the paper forms.

Version 1 was our first main iteration for testing

Summary of change in v1

Reason/source

Moved permit selection to start and give custom data about permit and customised flow for permit

Based on GDS ‘check before you start’ pattern. Allows users to find out correct rules and cost and also to customise the form based on the things the user needs to do.

Add payment options based on GOV.UK pay screens

Use of GOV.UK Pay platform instead of credit card form allows users to pay online

Remove ‘confirmations’ where these are implicit (eg for uploads)

Simplify for users

Remove multiple declarations and contact details for those declarations

Simplify for users - based on us seeking legal agreement to allow agent to declare on behalf of the operator.

Add done page and email confirmation

Standard patterns

Add mockup of GOV.UK guidance page

Shows likely journey as we won’t be allowed a start page due to volume of transactions

Version 1 to 2

Version 2 incorporates feedback from the first round of user testing.

Summary of change in v2

Reason/source

Move ‘harder to complete’ sections to start before easier to complete sections, such as about the operator.

V1 research - users don’t think they can complete in one go

For cheque and BACS we need to provide payment reminder on or after the 'done' page. - Cheque and BACS details added to done screen and email

User research and UX best practice - users may forget

Added permits that may not be available for online application - eg landspreading

User feedback - users were not sure why permits were not displayed.

Add 'special case' extra questions for 7 permits

Did not have time to add in v1

Link to full guidance for Fire Prevention Plan, not summary.

User feedback, and need to test reaction to full guidance

Technical competence: correct the guidance to explain there are other ways to meet the competence check.

Add indicative declaration text

Missing in v1

Add indicative data protection text

Missing in v1

Make site plan conditional for permits that don’t need them

User feedback

Change risk assessment text ‘Download and complete’ to ‘Check that you can comply with the…’

User feedback

Inconsistency between 5mb and 10mb upload limit - switch to 10MB

We have feedback on 5MB ‘straw man’ so switch to higher limit.

Use Data Returns multiple upload pattern for file uploads

User feedback - did not know they could upload more than one

Management system - wording around ‘Our management system does not meet your guidance’ - improve wording.

User and business feedback - this was a poor conversion from the paper form - still  needs further discussion.

Users are familiar with ‘BACS’

Version 2a to 3

Version 3 uses the newly-published GDS pattern for the task list.

Summary of change in v3

Reason/source

Replace the save and return code with version based on GDS examples. Note: 'Save and return' is not an official GDS pattern yet.

We plan that the save and return will use GOV.UK Notify.

Feedback from the data protection team - we can’t rely on all the data being made public so we need a higher standard of save and return authentication than the single code.

Replace the PAFS task pattern with the beta GDS ‘task list pages’ pattern

GDS service standard - use the GOV.UK design patterns

Version 3 to 4

Version 4 incorporates feedback from the third round of user testing.

Summary of change in v4

Reason/source

Change permit selection to an expanding list on one page and add feature to apply from the full list of standard rules permits.

Reduce the number of screens needed to choose a permit and demonstrate how permits could be applied for directly from the collection page list.

Save and return option added to the Task List page.

Feedback from user testing. Allow users to save and return, even if they choose not to when first asked.

Add 'What you need before you start' page.

Feedback from user testing. User would like to see in one place what they need in order to apply.

Removed installations and water categories

Currently not in scope.

Text changed to 'Complete offences statement' and 'Complete bankruptcy or insolvency statement'

Convictions and bankruptcy: users not filling in because they don't think it is applicable to them.

Version 4 to 5

Version 5 incorporates feedback from user testing and a GDS review of how we were using the task list pattern.

Summary of change in v5

Reason/source

Moved permit costs, time to wait after applying and 'what you need to apply' into task list.

Users can now check costs, time and the docs and info they need at any stage in the application process.

Replace 'Start a new application' launch page with 'Have you already started an application?'

No user need for the launch page as 'Apply for a standard waste permit' already selected on the previous page.

Change to Save and return pattern.

No longer optional. We will provide the means to return to an application later whether they use this feature or not. Moved to the start of the application process.

Upload certificate only displayed if you select a technical scheme.

Upload option brought into single screen.

Added text to links to external sites (opens new tab).

In user testing, opening new tabs was not clear for users unfamiliar with browser.

Move pre-app warning message to show if 'no' is selected and move to be near this option and only show if 'no' is selected.

Warning was too not needed at top for users who had had a pre-app discussion and wording was unclear to users.

Remove the 'Have you ordered a site location check?' page. Replace with link to order a site loation check on the 'read rules' page.

We discovered that every application has a new location screen done by P&SC - they don't look for or rely on any existing screens. To simplfy things for users, we don't ask for that information. Howeevr, we've added information to the read rules page to allow users to order a check, if they need it.

Replace details about location on the 'read rules page' with a link to allow user to order a location check.

Users do not understand how to make these location checks - eg "how do I know where boreholes are".

Simplify read rules page to remove extra weight limit and provide info about what to do if you don't comply.

Weight limit is only part of the rules. 'Comply with' not Plain English.

Change legal warning styling to panel

GDS standard is to use warning for legal warning. Our use was not correct.

Add WAMITAB qualifications to the upload screen

Allows users to find WAMITAB guidance on both 'What you need' and upload screens

Link directly to FPP guidance and remove short form of guidance

Short guidance did not test well with users as it was too short and longer detailed guidance has recently been updated so seem best to link to that. We need to test fire prevention plan guidance in beta.

Remove option to search on company name

Evidence from the flood risk project suggests that most users know how to find the company number. A number search will produce simpler results compared with a free text search (as few users will get the company name correct) so it will be easier to use in the end.

Add checkbox to allow user to say if they are an agent

Business need: I want to know if the contact for the application is an agent or consultant so that I can report on applications by agents and analyse use of the form by agents

Add individual journey

Allows us to test with individuals and sole traders

Add contact details to footer

Allows users to find a contact method on every if they are unable to complete the online process (AD).

Version 5 to 6

Version 6

Summary of change in v6

Reason/source

Added validation check that all tasks are complete. If not all complete, show a page listing the tasks to complete with a main action to return to the task list.

Some users feel that some tasks are optional. This page reminds them that all tasks need to be completed and lists the tasks to complete.

Change rules task title to 'Confirm that your operation meets the rules'

User testing of v5 showed that a couple of users regarded some tasks as optional. We want to test if the wording can change this perception.

Add text on done page to tell users that we'll contact them within 5 working days

Gives the user more clarity - we were able to add this after we received additional information from the business

Change 'save or resume' page text to 'Start a new application' / 'Open a saved application' from 'Have you already started an application?' Yes/No

Users were pausing to read this page carefully. New applicants were having to choose 'no' which was making them pause to think.

Swapped permit selection to a two-stage, category-first method. We combined 2 categories because they had few permits.

We tested the old categories against the new with 4 users. All users were able to find the correct permit with the new categories so we feel they are OK to take to further testing.

Removed BACS and cheque payment options so that debit/credit card payments are required for the cost of application

Business decision to remove other payment options in order to increase uptake of debit/credit card transactions.

Update drainage questions for SR2015-17/18 to a first draft for inital testing. The previous version was a copy of the paper form and was not complete.

This is a first draft for user lab testing based on a smart answer format.

Version 6 to 7

Version 7

Summary of change in v7

Reason/source

Set reference number to EA permit standard EPR/AB1234CD.

Make user lab feedback more realistic - request from researcher.

In HTML, add a 'rel="noopener noreferrer"' attribute to any links that use 'target="_blank"'.

GDS recommendation: this protects users against security vulnerabilities associated with the 'target="_blank"' attribute.

In task list, change 'Apply' text to 'Send and pay'.

Text was misleading to users - entire process is 'apply', this step in the task list is to send the application and pay.
