Version 12 to 13

Version 13

Summary of change in v13

User need, reason or source

Add screens to allow sole traders and individuals to apply for a permit. Add sole trader as an option so that we can capture their extra business details.

After companies, individuals are the next most common group of applicants (about 6% of applications). Defra stores customer records for businesses (in this case 'sole traders') separately from private individuals, so we designed the screens to allow users to identify themselves in a simple way and show them the required screens.

Replace the technical competence PDF with a Word and ODT version

During testing we found that applicants using Chrome browser to download and open the PDF form could not save the form and were uploading a blank form. We've replaced the PDF with Word and ODT versions that Chrome does not try to open.

Allow applicants for 'car breaker' - end of life vehicle - permits to check if their drainage system meets the rules

Users can't get a permit if they don't meet the rules and for these rule sets, the drainage requirements are not always spotted in the rule sets. We've broken out the requirements as options and explain if there is a problem. It's not an ideal method, but it makes the content clearer than a block of text.

Move Company Secretary email screens to a new single screen.

We had limited feedback that there's still some confusion about what this address is used for so we've rewritten the help text and moved the input to its own screen.

Add a feature to check cost and processing time

The Strategic Review of Charging introduced increased and new charges in April 2018. We had past feedback that users really valued having clear information about costs and with the new charges we felt there was a clear user need to know the cost of the application.

Add routing screens for permit holders that are not companies

The private beta service will work for companies but not for other applicant types. We want to make sure that other applicants are directed to the PDF form version of the service. Users told us that they find content on GOV.UK hard to navigate so we've given them links directly to the forms they need.

Add routing screens for rule sets that are not in the private beta. If users select a permit that we don't deal with in the service they are given a link to go directly to the correct forms.

We want to make sure that applicants are directed to correct method for applying if they cannot apply online.

Add a route for users who (incorrectly) end up on the service trying to apply for standard rules that we don't offer yet - for example flood risk.

We want to make sure that applicants are directed to correct method for applying if they cannot apply online.

Provide link to guidance for users who need to know how to identify the legal operator.

There is detailed guidance about this on GOV.UK so we provide a link to the correct guidance.

Implement a few small content changes to improve accessibility. For example, change the order of information in the 'check company' screen.

We carried out an accessibility review and implemented the changes to improve accessibility. This will be validated by an external review including users with access needs.

Minor updates to the technical competence screens.

Some terms used by the qualification awarding bodies had been changed and we had feedback from subject matter experts about minor updates to the conditions.

Add a screen to allow users to give details about the technically competent managers and the sites they supervise.

This requirement arrived late in the development process and we needed a tactical solution. We've provided a means to download a PDF form and upload it again. It's not ideal but seems to work OK.

Version 11 to 12

Version 12

Summary of change in v12

User need, reason or source

Added a simple save and return method into the service

There is a strong user need to be able to complete the application over a number of sessions. We've added a method based on an alpha GDS pattern that uses an email with a return link. We did have this before but it had been removed for the 'reduced' MVP.

Updated Bacs payment information

We added account name to the Bacs details. It's not required by the Bacs standard, but when we did a little research on typical bank payment screens, we discovered that it's an optional field. We felt it would be helpful to users to say what to use.

Added back links to 3 pages: check answers, payment and Bacs

These were added to have a consistent interface for users.

Update content following a content review by legal and subject matter experts.

These were added to have a consistent interface for users.

Version 10 to 11

Version 11

Summary of change in v11

User need, reason or source

Add BACS payment information to the final page and the application received email

We'd seen some evidence from earlier testing - where we added the BACS payment information to a separate page - showing some users did not click through to the final page. Therefore we've move the details to the final page and changed the continue button to 'Send application'.

Added GOV.UK Pay as the method for card payments instead of Worldpay.

GOV.UK Pay offers a better user experience and is more accessible than the Worldpay option.

Change done page to match new GDS pattern

The GDS pattern for the done page was updated to increase the heading size and margins.

Change use of term 'technical management' to 'technical competence'

Guidance on GOV.UK refers to 'technical competence' and so it's better for users to have a consistent use of terminology.

Version 9 to 10

Version 10

Summary of change in v10

User need, reason or source

v10 contains many big changes following a decision to build a slightly smaller MVP. We won't list all those changes here but these are the major changes.

Allow users to skip postcode entry for site without a postcode.

Some (rural) sites don't have a postcode and this causes problems for users.

Clarify help text for directors' date of birth.

A couple of users thought that this is a ‘security credential’ not to allow us to ‘perform’ a security check.

Adjust category names to add 'land treatment'.

A user pointed out that the land treatment text in storage hint was confusing and that adding the keyword to mobile plant would help select this category with confidence.

Remove save and return (for now)

We know users want this but we don't have enough information about the required software (Identity Management) to build this yet.

Add page to list everything that is needed for the application

Without save and return, users need to know up front what they need in order to apply.

Add page to allow users to apply using PDF forms, if their permit choice or applicant type is not available in the service.

Users need to know how to apply, even if they can't apply online initially because some permits are not covered by the service.

Remove cost page and move cost to what you need page.

Users like this page but it's not critical to a successful application so it has been de-prioritised for MVP.

Move management scheme information to the declaration section.

It's not critical to a successful application so it has been de-prioritised for MVP.

Move applicant (permit holder) type page to before the task list.

Allows non-company users to apply, even if they can't apply online initially because we will build first for limited companies only.

Clarify on the 'application sent' what happens after we receive the application. Remove reference to 5 days.

Users were slightly confused about the use of the term 'complete'. Users who had applied before (agents) knew the term 'duly made' and wondered if that was the same as 'complete'. The timescale was included becasue we thought it was helpful but users found it confusing.

Combine company name and address page.

The two items are for users to check and are not editable so it is a simplification to show them together in context. Testing shows that this performs OK.

Move Company Secretary email to the contact details screen.

When this was collected separately, and later, users were confused about which contact would get what information. We've put the inputs together and so far testing shows that this is clearer to users.

Move technical manager contact to after the qualification type pages.

When this was collected first, users were confused because they did not always have a name of someone with a qualification, if they were in the process of setting up a site and the person was 'getting' the qualification. Testing shows that users are happy with this new flow.

Move technical manager contact to after the 'qualification type' pages.

When this was collected first, users were confused because they did not always have a name of someone with a qualification, if they were in the process of setting up a site and the person was 'getting' the qualification. Testing shows that users are happy with this new flow.

Replace WAMITAB number with a certificate upload.

Our initial data suggested that most people (>98%) had a certificate. New data says that >40% will be 'getting a qualification' so we've simplified the screens to capture whatever evidence they need to provide.

Added ‘That’s not our registered office address’ help text to explain what to do if the company address is not correct

We heard one bit of evidence that some company owners may use a solicitor or company agent and had not since changed their address. As this blocks the application, this seems important to address.

Version 7 to 8

Version 8

Summary of change in v8

User need, reason or source

Swap card payment method to use Worldpay.

Business decision taken to use Worldpay initially and not GOV.UK Pay.

Allow users to choose to order a location screening check before they carry on with the application.

This has been added an an experiment to test in the next round of remote testing. It's been added because we know that users (about 250 a year) have a need to check their location before a full application (25% fail at this stage) and we know that some applications (5%) fail the location check after a full application.

Move save and return to be a separate task. It was triggered after the first task done.

Although testing showed that the save and return after a task worked OK, we noted that users were 'slightly surprised' or it was 'jarring'. This change is to test if a separate task will prevent this.

Remove 'main business address' and replace with a site contact.

The existing business systems don't store this address. We have a need to have a contact for the application, one for the site and one for invoicing, so that's what we've done.

Added a separate task to collect the invoice address. And added guidance to make clear what we need the address for. We're also allowing the user to select a previously entered address.

Invoice address is required for all permit holder types and the permit holder flow is already quite long so our hypothesis (to be tested), is that this will be clearer. We have evidence from P&SC that users are currently providing the wrong address because they think the address is needed for the application fee instead of the subsistence fee. Allowing the user to select an existing address makes it easier for them.

Added a screen to gather name and date of birth for the technically qualified manager.

There is a business need to perform enhanced conviction checks on this person, so we need to know who they are and their date of birth to ensure we are checking the right person.

Changed the WAMITAB evidence from a certificate upload to details about the qualified person.

The user does not have to scan their original certificate and continuing competence. EA now looks this up on the WAMITAB database.

Added a choice screen for the technically qualified manager to state what type of qualification the manager has.

The user only has to read the guidance appropriate to their qualifiation.

Added a new screen for the applicant to give an email address for company secretary or a director.

This means that applicants can get official notifications by email and not by post. This also help to save postage costs.

We've made a few updates to the wording following a review by another content designer.

This helps ensure we're meeting GDS standards.

Make the input box for director's date of birth slightly wider so it looks like a rectangle not a square.

On user assumed that the square box was a checkbox. Although this was only one user, it seems sensible to avoid replicating the appearance of a checkbox.

Added option to give different main UK business and billing addresses for limited companies.

A company permit holder may have a different main business address and/or billing address from the registered office address listed at Companies House.

Redesigned the directors' date of birth input screen to make it clearer that we only need the day date. We've also reduced the help text to cover the needs that we found in the lab.

In the Sheffield lab we saw people attempt to enter the whole date (eg '12/07/1956') instead of '12'. Users also mentioned having concerns about indentity theft and they wanted to know whey we needed the date of birth.

Add application reference number to the save and return 'check your email' screen.

Following the Sheffield lab we realised that it would be helpful for users to have the reference in case the email did not arrive. We already give the reference number in the email.

Add text: 'with or without treatment' to the 'Waste transfer station' permit category.

All three users in the Sheffield lab who were selecting rule sets in this category questioned if it "included treatment". We think this suggests that 'treatment' is a helpful keyword for this group of users.

Fix error in convictions help text and reduce complexity by showing only the correct text for each permit holder type.

Mistake in text spotted by both users and staff: "...or employees of a company” is not correct. The actual guidance is: ‘director, manager, secretary or similar officer’.

Make it clear that where there are two possible WAMITAB certifiates that either can be offered. Both are not required. We split the text into a list where there are two certificates to make it clearer and easier to read.

A user questioned this during the user lab.

Fix minor bugs in the select permit experimental pages.

Changes to the permit matrix had introduced bugs.

Version 6 to 7

Version 7

Summary of change in v7

User need, reason or source

Update mining waste plan content for SR 2009-8. Remove other questions for this permit.

Agreed with business to simplify requirements so that a mining waste plan check list is not needed.

Add these waste transfer station rule sets to the list of available permits: 2008-10 and 2008-11 inert waste; 2008-24 clinical waste and healthcare waste; 2008-25 clinical waste and healthcare waste treatment; 2009-5 inert and excavation waste under 250kte; 2009-6 inert and excavation waste transfer station with treatment under 250kte

These permits are available - the GOV.UK page was incorrect (it is being fixed soon).

Set reference number to EA permit standard EPR/AB1234CD.

Make user lab feedback more realistic - request from researcher.

Remove 'EPR/' from reference number so it reads 'AB1234CD'.

We are using this on save and return screens so we want to make it as short as possible. We are confident that even though there is a possibility of overlap with, for example, water permits, that our staff will be able to identify this from any enquiry as being a waste permit.

Allow user to give a trading name for company, partnership or sole trader.

Users who trade as 'Jane's Motors' may have a company called 123 Autos Ltd but hardly ever use that name. Collecting the trading name means that we can refer to their business using the term they most often use.

Added a timeout page.

The current content is just for the next user lab so we can use it to find out how users react to a timeout in the context of using the save and return function. The content is deliberately less helpful than it would be in the production code.

Collect the day date of birth for all company directors. We retrieve the list of active directors and their month of birth from Companies House. Users will have to supply the day date for each director in the list.

EA have new fraud detection checks in place. We need a date of birth to reliably match records when we do checks on directors.

Add help text for overseas companies to explain that they should register as an overseas company with Companies House before they apply for a permit.

This helps users representing overseas companies to understand the requirements for overseas businesses operating in the UK. The prior registration does add some extra work for the company, but it then simplifies the information they have to give us and allows us to make the checks we normally do.

Alter save and return journey to be triggered after the user starts a task in the application.

In the previous journey, users started on save and return. However, during testing we found that they had little recall of having done that so we want to test with a similar proceds, but at a different point in the user journey.

Add a 'page not found' page to show when users try to access a page that does not exist.

Helps users to work out what to do if they try to visit a URL that does not exist.

In HTML, add a 'rel="noopener noreferrer"' attribute to any links that use 'target="_blank"'.

GDS recommendation: this protects users against security vulnerabilities associated with the 'target="_blank"' attribute.

In task list, change 'Apply' text to 'Send and pay'.

Text was misleading to users - entire process is 'apply', this step in the task list is to send the application and pay.

Add inline validation to the task list.

Some users could not find the task that was not sone when they used the version where the validation message appeared on a separate page. In addition, inline validation is a better match to GDS patterns.

Add input to capture trading, business or company name for agents.

Business need to identify agents - some agents are known by their name, others by a company or business name.

Update permit categories to add 'deposit for recovery' and tweak labels slightly - converting 'recycling' to treatment and storage'.

Some users found the deposit for recovery tricky to find and it's a popular permit type so we think it justifies a category of its own. This is to be tested in August 2017 and reviewed after that.

Add a page to collect dates of birth for company directors.

Business requirement. We use this to check conviction and other records. To help users, we propose to get a list of the 'active' company directors from Companies House together with month and year. Users then will be asked to supply the day date for each director.

Add back links to most question pages.

User need is to get back to a previous page when they don't trust the back function in the browser. It's also a GDS standard introduced in July 2017: question pages should have a back link. We're working on adapting this to the task list pattern, but added it to the prototype to test later with users.

Add screen to allow users to download a copy of their application.

In the July user testing, several users said they'd like a copy of their application. In this version to be tested, we show it as a download after payment. Although some users suggested that it would be good to get by email, we're going to test without that becasue data protextion tell us that sending personal data by email may not be desirable.

Add permit details to confirmation email.

Users said they'd like the email to state the permit name.

Provide copyright information on postcode lookup pages.

We can't use the OS Places postcode lookup without adding this statement and a link to a page about terms of use. This is to make clear that the Open Gov licence does not also cover the OS Places data.
